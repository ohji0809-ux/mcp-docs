MCP 서버 연결하기

MCP 서버를 Claude Code에 추가하고, 연결 상태를 확인하고, 디스크에서 설정 파일을 찾는 방법을 알아봅니다.

Model Context Protocol(MCP)은 Claude Code가 이슈 트래커 검색, 데이터베이스 쿼리, 웹 브라우저 제어 등 기본 제공 도구 외의 기능을 사용할 수 있게 해줍니다. 이러한 도구들은 로컬 머신이나 호스팅 서비스에서 실행되는 MCP 서버에서 제공됩니다.

이 가이드에서는 Claude Code CLI로 MCP 서버를 처음부터 끝까지 연결하는 방법을 안내합니다. 완료하면 서버가 연결되어 응답하고, 디스크의 설정 위치를 파악하며, 가장 일반적인 연결 오류를 해결할 수 있습니다.

📌

데스크탑 앱, VS Code, 웹 등 다른 환경에서도 MCP 서버를 추가할 수 있습니다. 다른 환경에서 연결하기를 참고하세요.

시작하기 전에

다음이 준비되어 있는지 확인하세요:

  • Claude Code 설치 및 인증 완료
  • 프로젝트 디렉토리에서 터미널 열기 (빈 디렉토리도 가능)

서버 추가 및 확인

아래 예시는 Claude Code 문서 MCP 서버에 연결합니다. 이 서버는 Claude Code 문서 전체를 전문 검색할 수 있는 호스팅 서버로, 인증이나 특별한 설정이 필요 없어 연결 흐름을 처음 테스트하기에 적합합니다.

단계는 모든 서버에 동일합니다: 추가 → 연결 상태 확인 → 세션에서 사용, 마지막에 선택적 정리 단계가 있습니다.

1
MCP 서버 추가

서버를 Claude Code에 등록합니다. claude 세션 내부가 아닌 터미널에서 실행하세요. 대화를 시작하기 전에 서버를 설정하는 단계입니다.

bash 
claude mcp add --transport http claude-code-docs https://code.claude.com/docs/mcp

명령어 구성:

  • claude mcp add: Claude Code에 서버를 등록합니다.
  • --transport http: 로컬 프로세스가 아닌 URL에 호스팅된 서버임을 나타냅니다.
  • claude-code-docs: 직접 지정하는 이름입니다. docs라고 해도 동일하게 작동합니다.
  • https://code.claude.com/docs/mcp: 서버가 호스팅된 URL입니다.

명령 실행 후 Added HTTP MCP server claude-code-docs with URL: ...와 같은 확인 메시지가 출력됩니다. local config는 현재 프로젝트에 서버가 등록되었음을 의미합니다.

2
연결 상태 확인

서버 목록에 나타나는지 확인하고 상태를 점검합니다:

bash 
claude mcp list
상태 의미
✓ Connected 사용 준비 완료. claude-code-docs에서 이 상태가 보여야 합니다.
! Needs authentication 서버에 접근할 수 있지만 브라우저 로그인이나 토큰이 필요합니다.
✗ Failed to connect 서버가 응답하지 않습니다. 문제 해결을 참고하세요.
✗ Connection error 연결 시도 중 오류가 발생했습니다. 문제 해결을 참고하세요.
⏸ Pending approval 아직 승인하지 않은 프로젝트 범위 서버입니다.
3
서버 사용

세션을 시작하고 새 서버를 이름으로 사용하도록 요청합니다:

bash 
claude
text 
claude-code-docs 서버를 사용해서 MCP_TIMEOUT이 무엇인지 찾아줘
💡

Claude가 자체적으로 관련 도구를 선택하므로 보통 서버 이름을 프롬프트에 명시할 필요는 없습니다. 여기서는 데모 목적으로 명시합니다.

Claude가 처음 서버를 호출할 때 새 도구 사용 허가를 요청합니다. 승인하면 계속 진행됩니다.

4
서버 제거 (선택 사항)

실험이 끝나면 서버를 제거할 수 있습니다:

bash 
claude mcp remove claude-code-docs
📌

연결된 각 서버는 도구 이름과 서버 지침이 매 세션마다 로드되므로 Claude의 컨텍스트 창 공간을 차지합니다. 더 이상 사용하지 않는 서버를 제거하면 해당 공간을 확보할 수 있습니다.

서버 저장 위치

claude mcp add 명령은 서버 세부 정보를 설정 파일에 기록합니다. 기본적으로 local 범위에서 등록됩니다(현재 프로젝트에서만 활성화, 본인만 사용). --scope user로 모든 프로젝트에 등록하거나, --scope project로 팀원과 공유할 수 있습니다.

📌

claude mcp add는 PowerShell, Command Prompt를 포함한 모든 셸에서 동일하게 작동합니다. claude 세션 내에서는 /mcp 명령어로 이미 추가된 서버를 확인하고 관리할 수 있습니다.

디스크에서 설정 파일 찾기

claude mcp add 명령은 --scope 플래그에 따라 세 가지 범위 중 하나에 서버를 기록하며, 두 개의 파일에 걸쳐 저장됩니다.

범위 파일 적용 대상
local ~/.claude.json (프로젝트 항목 내) 본인만, 현재 프로젝트만 (기본값)
project 프로젝트 루트의 .mcp.json 프로젝트를 클론한 모든 사람
user ~/.claude.json (최상위 mcpServers 키) 본인만, 모든 프로젝트

Windows에서 ~/.claude.json은 %USERPROFILE%\.claude.json으로 해석됩니다. CLAUDE_CONFIG_DIR을 설정한 경우 해당 디렉토리 내의 .claude.json을 읽습니다.

claude mcp get claude-code-docs를 실행하면 어떤 범위에 서버 정의가 있는지 확인할 수 있습니다.

서버 범위 변경

서버 범위는 추가할 때 고정되므로 범위를 변경하려면 항목을 제거하고 새 범위로 다시 추가해야 합니다. 아래 두 경우 모두 먼저 로컬 항목을 제거합니다. 이미 제거했다면 이 명령을 건너뛰세요:

bash 
claude mcp remove claude-code-docs --scope local

모든 프로젝트에서 서버 사용

user 범위로 다시 추가하면 열어보는 모든 프로젝트에서 활성화됩니다(본인만 사용):

bash 
claude mcp add --scope user --transport http claude-code-docs https://code.claude.com/docs/mcp

팀과 서버 공유

project 범위로 다시 추가하면 프로젝트 루트의 .mcp.json에 기록됩니다:

bash 
claude mcp add --scope project --transport http claude-code-docs https://code.claude.com/docs/mcp

.mcp.json을 버전 관리에 커밋하세요. 저장소를 클론한 팀원들도 서버 승인 후 자동으로 연결됩니다.

추가 MCP 서버 예시

로컬 서버 추가

로컬 stdio 서버는 URL에 연결하는 대신 Claude Code가 머신에서 서브프로세스로 시작하는 프로그램입니다. 브라우저, 파일 시스템, 데이터베이스 소켓 같은 로컬 리소스에 접근이 필요한 도구에 사용합니다.

Playwright MCP 서버는 좋은 예시입니다. Claude에게 탐색, 클릭, 읽기가 가능한 브라우저를 제공하며 계정이 필요 없습니다. npx로 실행되므로 Node.js 18 이상이 필요합니다.

1
Playwright 서버 추가
bash 
claude mcp add playwright -- npx -y @playwright/mcp@latest

-- 구분자 뒤의 모든 것이 서버를 시작하기 위해 Claude Code가 실행하는 명령입니다. 다른 브라우저를 사용하려면 @playwright/mcp@latest 뒤에 --browser firefox를 추가하세요.

2
연결 확인
bash 
claude mcp list

처음 확인 시 npx가 패키지를 다운로드하는 동안 ✗ Failed to connect가 나타날 수 있습니다. 잠시 기다렸다가 다시 실행하세요.

3
브라우저 사용
text 
playwright를 사용해서 https://example.com을 열고 페이지 제목을 알려줘

브라우저 창이 열려 작동 과정을 확인할 수 있으며, Claude 출력의 도구 호출에 playwright 서버 이름과 browser_navigate 같은 작업이 표시됩니다.

로그인이 필요한 서버 연결

Sentry, Linear, Notion 같은 호스팅 서비스는 OAuth 뒤에 MCP 서버를 운영합니다. 서버 URL을 추가한 후 브라우저로 로그인합니다.

1
서버 추가
bash 
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

추가 후 claude mcp list에서 ! Needs authentication이 표시됩니다. 다음 단계에서 로그인을 완료하세요.

2
브라우저에서 인증

Claude Code 세션을 시작하고 MCP 패널을 엽니다:

text 
/mcp

목록에서 sentry를 선택하고 Enter를 눌러 Authenticate를 선택합니다. 브라우저가 Sentry 로그인 페이지로 열립니다. 연결을 승인하면 서버 상태가 연결됨으로 변경됩니다.

3
서버 사용

내가 접근할 수 있는 Sentry 프로젝트가 뭐야?와 같이 서비스가 필요한 질문을 하면 출력에 sentry 서버 이름이 표시된 도구 호출을 확인할 수 있습니다.

.mcp.json 직접 편집

모든 범위 파일은 서버 항목에 동일한 JSON 형식을 사용합니다. 아래 예시는 이 가이드의 두 서버를 모두 정의합니다.

프로젝트 루트에 .mcp.json을 만드세요:

json 
{
  "mcpServers": {
    "claude-code-docs": {
      "type": "http",
      "url": "https://code.claude.com/docs/mcp"
    },
    "playwright": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    }
  }
}
  • HTTP 서버의 경우 url이 Claude Code가 연결하는 엔드포인트입니다.
  • stdio 서버의 경우 command와 args가 실행할 프로그램입니다.

파일을 저장한 후 프로젝트에서 새 Claude Code 세션을 시작합니다. Claude Code는 시작 시 .mcp.json을 읽습니다. 처음 보는 프로젝트 범위 서버는 승인 프롬프트가 표시됩니다.

다른 환경에서 연결

이 가이드는 claude mcp CLI 명령을 사용하지만, 모든 Claude Code 환경에서 MCP 서버를 연결할 수 있습니다:

  • Claude Code 데스크탑 앱: Connectors UI를 통해 서버를 추가합니다.
  • Claude Desktop 채팅 앱: claude mcp add-from-claude-desktop으로 claude_desktop_config.json의 서버를 가져올 수 있습니다 (macOS 또는 WSL).
  • VS Code: MCP로 외부 도구 연결을 참고하세요.
  • 웹의 Claude Code: 저장소의 .mcp.json을 읽습니다.
  • Claude.ai: claude.ai/customize/connectors에서 추가한 커넥터가 CLI에 자동으로 로드됩니다.

문제 해결

서버가 연결되지 않는 경우 세션 내 /mcp 또는 셸에서 claude mcp list로 상태를 확인하고 아래 증상을 매칭해 보세요.

Claude Code가 현재 디렉토리의 서버를 찾지 못했습니다. 일반적인 원인:

  • 다른 프로젝트에서 claude mcp add를 실행했습니다. Local 범위 서버는 추가한 프로젝트에만 연결됩니다. 현재 있는 프로젝트에서 다시 추가하거나 --scope user로 추가하세요.
  • 잘못된 경로의 설정 파일을 편집했습니다. 올바른 파일은 ~/.claude.json과 <project>/.mcp.json입니다.

HTTP 서버의 경우 URL이 머신에서 접근 가능한지 확인합니다:

bash 
curl -I https://mcp.sentry.dev/mcp
  • 404 또는 405: 서버는 정상입니다. MCP 엔드포인트는 POST 요청에만 응답합니다.
  • 401 또는 403: 서버가 정상이고 인증이 필요합니다.
  • 응답 없음: URL과 네트워크를 확인하세요.

stdio 서버의 경우 터미널에서 직접 명령을 실행해 오류를 확인합니다:

bash 
npx -y @playwright/mcp@latest

서버가 기본 30초 시작 시간 초과를 초과했습니다. MCP_TIMEOUT 환경 변수로 제한을 늘리세요 (밀리초):

bash 
MCP_TIMEOUT=60000 claude

PowerShell에서는:

powershell 
$env:MCP_TIMEOUT = "60000"; claude

같은 범위에 동일한 이름의 서버가 이미 있습니다. 기존 항목을 먼저 제거하거나 다른 이름을 선택하세요:

bash 
claude mcp remove claude-code-docs

여러 범위에 이름이 존재하면 --scope로 삭제할 복사본을 지정하세요.

세션 내 /mcp를 실행하고 서버를 선택하여 도구 목록을 확인하세요. 목록이 비어 있으면 서버가 시작은 됐지만 도구를 등록하지 않은 것으로, 주로 API 키 같은 필수 환경 변수가 누락되어 있습니다.

claude mcp add에서 --env KEY=value로 변수를 전달하거나 .mcp.json 항목의 env 필드에 지정하세요.

Claude Code는 세션 시작 시 .mcp.json을 읽습니다. 파일 편집 후 세션을 종료하고 재시작하세요.

이전에 서버를 거부했다면 프로젝트 승인을 초기화하세요:

bash 
claude mcp reset-project-choices

/mcp를 실행하고 서버를 선택한 후 Authenticate를 다시 선택하세요. 브라우저가 자동으로 열리지 않으면 터미널에 표시된 URL을 복사하여 수동으로 열어보세요.
